1 Build Vulnerable Target
The Kyber library source code is constant-time — it shouldn't leak secrets via timing.

But when Clang 15-18 compiles it with the -Os optimization flag, it introduces a secret-dependent branch in poly_frommsg().

The Makefile patches -O3 to -Os and compiles with Clang to trigger this bug.
Makefile
sed -i 's/-O3/-Os/g' kyber/ref/Makefile CC=clang make -C kyber/ref shared # This produces: # lib/libpqcrystals_kyber512_ref.so # compiled with -Os (the vulnerable flag)
What -Os changes
Source code (safe): r = ct >> 15; m = r - 1; Clang -Os compiles this into: branch if bit == 0 goto L1 The branch takes different time depending on the secret bit — this is the side channel.

6 Outlier Filter + Mean & Variance Update
If the differential is too large (context switch, interrupt), discard it.

Otherwise, update the running mean and variance for this CT using Welford's online algorithm. No need to store all measurements — just the running mean, count, and M2 accumulator.

The variance is used later to assess per-bit confidence — if a CT's mean is ambiguously positioned between the two clusters, the guess is rejected and more iterations are collected.

Classification is gated: at least 500 samples per CT must be collected before any guess is attempted, preventing premature decisions on noisy data.
attack.c : 159-170
// outlier filter if (fabs(diff) < 2000) { // Welford's mean + variance delta = diff - means[ct_index]; means[ct_index] += delta / (double)(++nbs[ct_index]); delta2 = diff - means[ct_index]; m2s[ct_index] += delta * delta2; } else { discarded++; } // gate: need ≥500 samples per CT min_nbs = min_of(nbs[0..8]); if (min_nbs ≥ 500) evaluate();

8 Per-bit Confidence + Classification
First, compute the anchor gap: the distance between the two known anchors (CT[7] and CT[8]).

For each of the 7 attack CTs, measure distance to both anchor means. Before classifying, apply a per-bit confidence check:

If the nearer distance exceeds 30% of the anchor gap, the bit is ambiguous — reject the entire guess and collect more iterations.

Only if all 7 bits pass the confidence check does classification proceed: the closer anchor wins.
guess.h : 74-93
// anchor gap for confidence anchor_gap = fabs(means[7] - means[8]); for (ct_index=0; ct_index < 7; ct_index++) { // distance to each anchor dist_zero = fabs(means[7] - means[ct_index]); dist_one = fabs(means[8] - means[ct_index]); // per-bit confidence check near_dist = min(dist_zero, dist_one); if (near_dist > 0.3 * anchor_gap) return DONT_KNOW; // ambiguous! // classify: closer anchor wins m_guesses[ct_index] = (dist_zero > dist_one) ? 1 : 0; }

10 Lookup Table → Coefficient
Scan the 7-row lookup table to find a matching pattern.

Each row corresponds to a coefficient value. The match at row i=4 is then converted using the polarity formula:

For non-zero index positions: coeff = 3 - i
For index 0: coeff = i - 3

This accounts for the NTT transform's effect on sign.

11 Confidence Check + Result
Every 512 iterations, the guess is re-evaluated. If it matches the previous guess, confidence increments. If it changes, confidence resets to 0.

After 20 consecutive stable evaluations (CONFIDENCE_HIGH), the guess is accepted with high certainty.

If 100,000 iterations pass without reaching confidence 5 (CONFIDENCE_MEH), the loop continues beyond the default limit until at least that minimum confidence is achieved.

This coefficient is done. The attack moves to s[43] and repeats the entire process. After all 512 coefficients are recovered, the full secret key is known.
